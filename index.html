01 · Overview
What it does
While you scroll

Reelio's Accessibility Service silently captures scroll velocity, dwell time, swipe completion ratios, back-scroll rewatches, exit attempts, ambient light, accelerometer variance, audio routing, previous app context, and dozens more signals — all in real-time, per reel.

After you close Instagram

The ALSE model runs on-device via Chaquopy, classifying each reel as CASUAL (passive browsing) or DOOM (compulsive capture). The model learns your personal baseline over time, growing more accurate with each session.

04 · The Brain
ALSE Model

The Adaptive Latent State Engine lives in reelio_alse.py and runs entirely on-device via Chaquopy. No server dependency. No cold API call. Nine architectural pillars working in concert.

01
Personalized Bayesian Baseline
HMM emission parameters are initialized from your rolling history, not global defaults. Your personal mean and variance per feature define the prior — making the model yours from session one.
02
Self-Calibrating Emission Weights
After each session, KL-divergence between CASUAL and DOOM emission distributions is computed per feature. Features that better separate the two states gain higher weight. The feature_weights vector is persisted and surfaced in the UI.
03
Hierarchical Temporal Memory
Three memory banks: recent (5 sessions), medium (20), long (50). The model interpolates between them based on data density, preventing both short-term overfitting and long-term staleness.
04
Continuous-Time Markov Chain
Session gaps are modeled using a CTMC with matrix exponential. Longer gaps revert the transition matrix toward the stationary distribution — accurately modeling the "cooling off" effect between sessions.
05
Survival Framing
The model fits a geometric hazard rate per state — modeling how likely you are to stop scrolling at each reel. DOOM has a dramatically lower hazard rate. You keep going.
06
Regime Change Detector
Tracks KL-divergence between recent behavior and stored baseline. If it exceeds a threshold (life event, holiday, new phone), the long-term memory bank is frozen — preventing drift corruption.
07
Sparse-Data Guard
model_confidence = min(1.0, n_sessions / 20). Below 20 sessions, inferences are blended with the prior proportionally. The UI shows this value explicitly so you always know how much to trust the model.
08
Contextual State Priors
A lightweight logistic regression runs at session start using 4 physical context features (time of day, charging state, previous app category, ambient light) to set the initial state probability — before any reel evidence is seen.
09
Composite Doom Score
A fully interpretable, model-free heuristic (0–100) with 7 named components. Decoupled from the HMM's latent probability so users can understand exactly why a session was flagged, in plain terms.

08 · Privacy
Zero data
leaves your device.

No API calls. No analytics SDKs. No crash reporters. No advertising IDs. The only network permission is for loading WebView fonts — and even that can be removed by bundling fonts locally.

All CSV data written to getExternalFilesDir(null) — app-specific storage, inaccessible without root.
Room DB and model state stored in internal storage. Inaccessible to any other app without root.
Export raw CSV via the app's Export button. Delete everything with the Clear Data button.
Reelio never reads, stores, or transmits the content of any Instagram post, caption, username, or media.
09 · Honesty
Known limitations
  • ⚠
    Accessibility Service detection drift Instagram occasionally updates its UI layout, changing the accessibility event tree. Reel boundary detection (via TYPE_WINDOW_CONTENT_CHANGED) may occasionally double-count or miss a reel transition.
  • ⚠
    Audio track identity is approximate UniqueAudioCount uses accessibility content descriptions as a proxy — not actual audio fingerprinting. It's an estimate.
  • ⚠
    Repeat content detection is a placeholder RepeatContentFlag and ContentRepeatRate are currently zeros. Full implementation requires content fingerprinting, which is on the roadmap.
  • ⚠
    MicroMovementRms is unimplemented Currently logged as 0f. True RMS micro-movement requires integrating raw accelerometer over reel duration — the performance overhead tradeoff is being evaluated.
  • ⚠
    Python cold start latency The first inference per app session takes 2–4 seconds due to Chaquopy Python initialization. All subsequent inferences in the same session are near-instant.
